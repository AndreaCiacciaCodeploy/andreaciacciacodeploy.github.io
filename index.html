Ogni giorno, in automatico, una news viene letta e riassunta da chatgpt3.
Per realizzare questa pagina ho lavorato sulle seguenti componenti:

JAM

  • static site generator. Strumento che permette di creare pagine web statice. Per questo sito ho utilizzato HUGO.
  • headless cms. Sistema di gestione dei contenuti in cui il back-end e i dati sono separati dal frontend. I contenuti di un headless cms sono gestiti tramite API. Per questo sito (ma solo per alcuni contenuti) ho utilizzato FORESTRY .

NEWS & AI

  • news api. Scelgo la news da riassumere tramite l’utilizzo di un servizio di news api.
    Questo -> NEWSAPI
  • OPENAI. Il pezzo forte. Il sistema di AI di OpenAI che permette di fare svariate operazioni. Lo uso per ottenere un riassunto del testo di interesse ottenuto dalla specifica news.

INFRASTRUCTURE

  • heroku. Provider di servizi cloud. Qui ho deployato una piccola app in python che implementa l’integrazione di NEWS e AI indicati in precedenza. Lo trovi QUI
  • github pages. Servizio di github per l’hosting di siti e pagine web statiche. Lo trovi QUI

AUTOMATION

  • github actions. Azioni automatiche che vengono eseguite in relazione a uno specifico evento sul repository github. In questo caso quando viene pushato un file .md con il contenuto del post, viene avviata la pipeline che effettua la build di hugo e deploya il sito su github pages.
  • heroku deploy. Deploy automatico dell’app python dopo ogni push inviato al repository.
  • cron-jobs Servizio che permette di impostare cron e scheduler. Ho impostato un’operazione giornaliera che effettua la chiamata api all’app python su heroku che avvia lo script di gestione del contenuto. cron-jobs.org
