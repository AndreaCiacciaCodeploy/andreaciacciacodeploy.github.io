JAM stack

2022-11-12
frontend , javascript , jam

Jamstack is an architecture designed to make the web faster, more secure, and easier to scale. It builds on many of the tools and workflows which developers love, and which bring maximum productivity. – Jamstack.org

JAM stack non è una tecnologia, ma uno stack di diverse componenti per la creazione di siti web. L’acronimo JAM identifica:

● Javascript

● Api

● Markup

Jamstack rappresenta la volontà di realizzare siti web statici ottenendo vantaggi in termini di performance, velocità, sicurezza, scalabilità e flessibilità.

JAM è basato su due concetti fondamentali:

● pre-rendering. Ossia la creazione di file statici (html, css, js) che possono quindi essere hostati direttamente all’interno di CDN e senza necessità di web server per l’ottenimento delle risorse richieste dall’utente.

● disaccoppiamento. Ossia la separazione delle componenti applicative (frontend,backend) rendendo i diversi layer autonomi e indipendenti.

Per realizzare un sito web con stack JAM sono necessari i seguenti componenti:

● static site generator (SSG)

● headless cms (non obbligatorio)

● static host website o cdn

Realizzare un’app in SPA permette di lavorare già in ottica JAM, tuttavia in questa modalità non è previsto il pre-rendering delle pagine che vengono invece elaborate dalla SPA tramite JS appesantendo quindi le performance lato browser (e quindi lato client/utente finale).

Dalla nascita dello stack JAM, sono nati diversi servizi di SSG e headless cms.

Tra questi ne sono nati alcuni per la generazione di siti web statici partendo da SPA realizzate in Angular.

Tra questi troviamo Scully (https://scully.io/) uno dei più consigliati in rete.

Scully (come altri SSG come per esempio Hugo) permettono di generare file statici partendo da una SPA e di integrare componenti per la creazione (per esempio) di blog e contenuti.

Per migliorare la gestione dei contenuti è successivamente possibile utilizzare headless cms come Forestry.io che si integra perfettamente in applicativi JAM.

● trasformare la SPA in un applicativo JAM mantenendo tuttavia lo sviluppo impostato (cambia solo la fase di build).

● Scully (al posto di hugo) permette un’integrazione nativa di contenuti blog all’interno della SPA Angular. Diversamente occorrerebbe integrare logiche di routing tra angular app e altre pagine esterne (ma sempre dentro lo stesso host).

● Dopo l’integrazione di scully, sarà possibile integrare forestry per la gestione dei contenuti (creando a sua volta file statici .html).

Questa metodologia risponde ai requisiti introdotti all’inizio del testo. La trasformazione della SPA in contenuto statico velocizza il sito web e lo rende seo friendly. I contenuti dinamici verranno gestiti tramite API tramite JS.
